# Step 7: 生成测试数据

**生成 25 组测试数据 → 只编辑 `work/mkin.h` 的 `test()` 函数**

## 测试数据要求

生成 25 组测试数据，覆盖以下类型（**无分组，每点独立计分**）：

| 用例编号 | 类型 | 说明 |
|---------|------|------|
| 1-2 | 样例数据 | 直接复制题目样例 |
| 3-8 | 小规模 + 特殊性质 | 验证基本功能 |
| 9-11 | Hack 数据 | 针对常见错误写法 |
| 12-20 | 中大规模 | 验证效率 |
| 21-25 | 随机回归 | 覆盖各种情况 |

**修改 `test()` 后，必须同步更新：**
1. `mkin.h` 顶部的 `SUBTASKS[]` 数组（仅作注释标记用）
2. `testdata/config.yaml` 无需更新（OJ 自动从文件名推断）
3. 25 组测试点总分 100（每点约 4 分）

## 特殊性质数据示例

根据题目的不同，特殊性质可能包括：

| 性质 | 说明 | 针对的错误 |
|------|------|-----------|
| 单调性 | 输入有序（递增/递减） | 错误排序算法 |
| 所有值相同 | 数组所有元素相等 | 未处理重复值 |
| 二分图性质 | 特定图结构 | 错误图算法 |
| 素数密集 | 大量素数 | 错误素数判断 |
| 极端值 | 最大/最小可能值 | 溢出错误 |

## Hack 数据设计

针对常见错误写法设计数据：

| 常见错误 | Hack 数据特征 |
|---------|--------------|
| int 溢出 | 使用接近限制的大数 |
| 边界漏判 | N=0, N=1, N=max 等 |
| 精度错误 | 需要精确计算的小数 |
| 超时错误 | 迫使 O(n²) 算法超时 |
| 错误贪心 | 使贪心策略失败的数据 |

## 配置文件

`work/testdata/config.yaml` 使用最简格式（OJ 自动从文件名 `1.in`~`25.in` 推断测试点）：

```yaml
type: default
time: 1s
memory: 256m
```

> **说明**：不写 `subtasks` 或 `cases`，OJ 会根据文件名（`1.in`, `1.out`, ...）自动识别 25 个测试点，每点独立计分，总分均摊。

## 编译运行

```bash
cd work
g++ -o mkdata mkdata.cpp -std=c++17
./mkdata
```

## 大样例处理

| 大小 | 处理 |
|------|------|
| < 500 字节 | `read_file` 读取 |
| ≥ 500 字节 | 禁止 `read_file`，用 shell 验证 |
